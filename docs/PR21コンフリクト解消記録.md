## 解消内容

### `Assets/Scripts/KingHealth.cs`

PR側には次の変更があった。

- 最大HPを `30000` に変更
- 初期化処理を `InitializeHealth` に集約
- HP回復処理 `Heal`
- 最大HP比率による回復処理 `HealByMaxHealthFraction`
- `UIHandler.instance` が存在しない場合も安全なHPバー更新
- `CurrentHealth` と `MaxHealth` プロパティ
- `sunDamage = 1`

main側には、リザルト画面の `HPBarResult` から参照するための次のAPIがあった。

```csharp
public static int hp;
public static int maxHp;
```

片方だけを採用せず、次のように統合した。

- PR側の初期化、ダメージ、回復、UI更新処理を維持
- 最大HPはPR側の `30000` を維持
- `hp` と `maxHp` をstaticにしてmain側のリザルト表示との互換性を維持
- `sunDamage = 1` を維持
- 不要になった `UnityEngine.UIElements`、`sunTimer`、`sunInterval` は採用しない

これにより、PR側のアイテム回復処理とmain側のリザルト画面HP参照を両立させた。
