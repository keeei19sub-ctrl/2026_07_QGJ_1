# UIアイデア出し用プロンプト

以下をそのままLLMへ貼り付けて使用する。

---

あなたはゲームUI/UXデザイナーです。以下のゲーム概要と現状UIをもとに、ゲームプレイの視認性を高めつつ、画面の世界観を統一するUI改善案を提案してください。

## ゲーム概要

- 2Dの見下ろし型ゲーム。
- プレイヤーは傘を持って移動し、同行する王を日差しや左右から飛んでくる投石から守る。
- 王は店を巡りながら画面下側から上側のゴールへ進む。
- プレイヤーは店でアイテムを購入し、選択中のアイテムを使用できる。
- 主に伝えるべき情報は、王のHP、ゴールまでの進行度、飛来物の方向、王が画面外にいる方向、店の商品、所持アイテムである。
- 現在のUIはUnity UI Toolkitで実装され、基準解像度は1980×1080。各UIの位置と大きさは画面に対する割合で指定する。

## 現在の画面レイアウト

座標は画面左上を `(0%, 0%)`、右下を `(100%, 100%)` とする。

```text
┌──────────────────────────────────────────────────────────────────┐
│ [王HP: 左上の50%×50%領域]       [王が上にいる場合の方向表示]     │
│                                                                  │
│ [縦型                                                            │
│  進行度]                                      [店の商品パネル]   │
│  左3%                                         右側25%            │
│  上15%                   ゲーム画面             店の近くのみ       │
│  幅8%                                                            │
│  高70%      [左からの警告]          [右からの警告]               │
│                                                [購入結果メッセージ]│
│                                                右25%・上57%       │
│                                                [所持アイテム]      │
│ [王が下にいる場合の方向表示]                   右下15%×35%        │
└──────────────────────────────────────────────────────────────────┘
```

## UI別の現状

### 1. 王のHPゲージ

- 目的: 守る対象である王の残りHPを示す。
- 表示内容: 現在HP ÷ 最大HP。数値表記はない。
- 表示位置: 左上を起点とした幅50%、高さ50%の領域。
- 内部位置: 外枠内の左41%、上34%、幅56%、高さ28%に横ゲージを配置。
- 見た目: 専用の装飾フレーム画像とゲージ画像。
- 表示タイミング: 常時表示。ダメージと回復時に更新。
- 現状の問題:
  - 画面の50%×50%を占める定義で、実際のゲージ情報量に対して領域が大きい。
  - Flexレイアウトと横幅による残量制御が競合し、正しく縮まない可能性がある。
  - 王のHPであることを示す名前、顔アイコン、数値などはない。

### 2. ゴールまでの進行度ゲージ

- 目的: 王が開始地点からゴールまでどれだけ進んだかを示す。
- 表示内容: 王のY座標を基準にした0～100%の進行率。数値表記はない。
- 表示位置: 左3%、上15%、幅8%、高さ70%。
- 見た目: 縦長の装飾フレームを赤い矩形が下から上へ満たす。
- 表示タイミング: 常時表示し、毎フレーム更新。
- 現状の問題:
  - HPゲージと同じ左側にあり、視覚的な競合が起こりうる。
  - 赤色はHPや危険を連想しやすく、進行度との意味の切り分けが弱い。
  - スタート、途中の店、ゴールなどの目印はない。

### 3. 選択中・所持アイテム表示

- 目的: 現在使用できるアイテムと所持数、効果を示す。
- 表示位置: 右下、幅15%、高さ35%。
- 表示構成:
  - 上半分: アイテム画像。
  - 中央15%: アイテム名と所持数。
  - 下35%: 説明文。
- 見た目: 半透明の黒背景、白い名前欄、薄い黄緑の説明欄。
- 未所持時の表示:
  - `アイテムなし`
  - `×0`
  - `アイテムを入手すると、ここに説明が表示されます`
  - 共通アイテム画像を不透明度25%で表示。
- 所持時の表示:
  - 選択中アイテム名
  - `×所持数`
  - アイテムの効果説明
- 表示タイミング: 常時表示。
- 現状の問題:
  - すべてのアイテムで同じ画像を使っている。
  - 説明文が常時大きな領域を取り、ゲームプレイ領域を圧迫する。
  - 使用キーが説明文の中にしかなく、瞬時に把握しづらい。
  - アイテム効果の残り時間を表示するUIはない。

現在のアイテムは次の3種類。

| アイテム | 価格 | 使用効果 |
| --- | ---: | --- |
| 王の薬 | 30 G | Shiftキーで使用。王のHPを最大HPの10%回復する。 |
| 大傘の薬 | 50 G | Shiftキーで使用。10秒間、傘と防御範囲を2倍にする。 |
| 投石止めのお守り | 70 G | Shiftキーで使用。10秒間、すべての投石を停止する。 |

### 4. 店の商品パネル

- 目的: プレイヤーが近くにいる店の商品を提示する。
- 表示位置: 右端の幅25%、高さ100%の領域内。上5%、下25%、右5%の余白がある。
- 表示構成: 背景画像、商品名、価格、商品説明。
- 表示タイミング: プレイヤーが店の範囲内にいる間だけ表示。
- 現在表示できる内容:
  - 商品名
  - `価格 G`
  - 商品未設定時は `商品未設定` と `--`
- 購入操作: Spaceキー。
- 現状の問題:
  - 商品説明はデータと接続されておらず、英語の仮文字列 `Description` のまま。
  - 購入キーの案内が表示されない。
  - 所持金表示用のコードはあるが、画面要素が存在しないため所持金が表示されない。
  - 右下の所持アイテムUIと表示領域が近く、情報が集中している。

### 5. 購入結果メッセージ

- 目的: 購入操作の成功・失敗理由を伝える。
- 表示位置: 右端、上57%、幅25%、高さ7%。
- 見た目: 黒背景に白文字。
- 表示しうる文言:
  - `購入しました`
  - `お金が足りません`
  - `アイテムは1個しか持てません`
  - `商品が設定されていません`
  - `購入できません`
  - `購入に失敗しました`
- 現状の問題:
  - 通常時は文字だけ空になり、黒い帯は常時残る。
  - 成功、警告、失敗が同じ配色と見た目で、判別しにくい。
  - 自動的に消える演出、アイコン、アニメーションなどはない。

### 6. 画面外にいる王の方向表示

- 目的: 守る対象の王がカメラの上下へ外れた際、居場所を知らせる。
- 表示位置:
  - 王が上にいる場合: 画面上端1%。
  - 王が下にいる場合: 画面下端1%。
  - 横位置は王の位置に追従し、左右端から最低1%空ける。
- 大きさ: 幅5%、高さ9%。
- 見た目: 黄色い三角形。
- 表示タイミング: 王全体が上下方向へ画面外に出たときだけ表示。
- 現状の問題:
  - 三角形だけで、王を示す情報や距離はない。
  - 左右方向へ画面外に出た場合の専用表示はない。

### 7. 飛来物の左右警告

- 目的: 左右のどちらから投石が飛んでくるかを事前に知らせる。
- 表示位置:
  - 左警告: 左0%、上29%、幅16%、高さ42%。
  - 右警告: 右0%、上29%、幅16%、高さ42%。
- 見た目: 薄赤色に着色した王の画像。右側は左右反転。
- 表示タイミング: 飛来物が出現する0.8秒前から出現直前まで。
- 現状の問題:
  - 王の画像を警告に流用しており、飛来物警告だと直感的に伝わりにくい。
  - 画面の16%×42%と大きく、ゲーム画面をかなり隠す。
  - 警告時間が短いため、形・色・動き・音を含む即時認識が重要。

## 現在存在しないUI

- 操作説明
- ポーズ画面
- ゲームオーバー画面
- リザルト画面
- 所持金表示
- アイテム効果の残り時間表示

## UI改善で重視したいこと

- プレイヤーの主な注視対象は、ゲーム画面内の王、傘、飛来物である。UIがプレイ領域を隠しすぎないこと。
- 最優先情報は「王の危険」「飛来物の方向」「王のHP」。次に「進行度」「使用可能アイテム」「店情報」。
- 王に関係する情報、危険に関係する情報、買い物に関係する情報を、色・形・配置で明確に分類すること。
- 常時表示する情報と、一時的に出す情報を整理すること。
- 日本語テキストが長くなっても破綻しにくいこと。
- 16:9を基本としつつ、各位置・サイズは画面比率（%）で提案すること。
- 実装負荷を考慮し、Unity UI Toolkitで作りやすい案にすること。

## 提案してほしい内容

次の形式で、方向性の異なるUI案を3案提案してください。

1. 各案のコンセプトと狙い
2. 全UIの再配置案。各要素について左・上・幅・高さを%で示す
3. 常時表示、一時表示、非表示を切り替えるルール
4. 色、形、背景、アイコン、文字サイズ、余白の方針
5. HP減少、投石警告、購入結果、アイテム効果中に使うアニメーション案
6. 現在不足しているUIのうち、追加すべきものと優先順位
7. Unity UI Toolkitでの実装難易度を「低・中・高」で評価
8. 最後に3案を比較し、このゲームに最も適する案を1つ推薦する

必要であれば、各案の画面構成をASCIIワイヤーフレームでも示してください。単に見栄えを良くするだけでなく、プレイ中の判断速度と視線移動の少なさを重視してください。
