# アイテム実装計画

## 目的

購入済みアイテムを`Shift`キーで使用し、以下の3種類の効果を発動できるようにする。

- 王のHPを回復する
- 傘を大きくして防御範囲を広げる
- 一定時間、投石を止める

## アイテム仕様

| アイテム名（仮） | 価格（仮） | 効果 |
| --- | ---: | --- |
| 王の薬 | 30G | 王のHPを最大HPの10%回復する |
| 大傘の薬 | 50G | 10秒間、傘の見た目と判定を2倍にする |
| 投石止めのお守り | 70G | 10秒間、投石を完全に停止する |

HPが満タンの状態で王の薬を使用した場合も、アイテムは消費する。

## 店舗配置

- 現在生成される14店舗は維持する。
- 道中に分散した6店舗だけ商品を販売する。
- 3種類のアイテムを、それぞれ2店舗で販売する。
- 残り8店舗は商品を設定しない空店舗にする。
- 販売店舗の位置と商品配置は`ShopManager`のInspector設定から調整可能にする。
- 商品がない店舗では、購入できないことが分かる表示を出す。

## データ設計

`ItemDefinition`へ以下の情報を追加する。

- アイテム効果種別
- 効果量
- 効果時間

既存の14個の仮アイテムは3種類の正式な仮データへ置き換える。`ShopManager`の検証処理も、店舗数と同数の固有アイテムを要求せず、販売に必要な3種類が登録されていることを確認する形へ変更する。

## 使用処理

プレイヤーへアイテム効果を振り分ける専用コンポーネントを追加する。処理順は以下とする。

1. `Shift`キーの入力を検出する。
2. 選択中のアイテムを取得する。
3. 効果種別に応じた処理を実行する。
4. 効果の発動に成功した場合、所持アイテムを1個消費する。
5. インベントリUIを更新する。

コード上の既定キーとプレイヤープレハブの入力設定は、どちらも`Shift`へ統一する。

## 王のHP回復

- `KingHealth`へ回復専用APIを追加する。
- 回復量は最大HPの10%とする。
- 現在の最大HPが30000の場合、回復量は3000となる。
- 回復後のHPは最大HPを超えないようにする。
- 使用直後にHPバーへ反映する。

## 傘の拡大

- 効果時間は10秒とする。
- 傘の表示サイズを通常時の2倍にする。
- 左右の投石防御判定を通常時の2倍にする。
- 王を日差しから守る影判定も表示に合わせて拡大する。
- 効果開始前のTransformとCollider2Dの値を保存し、終了時に正確に復元する。
- 効果中に同じアイテムを再使用しても4倍にはせず、2倍のまま残り時間を10秒へ更新する。
- プレイヤーが無効化された場合も、拡大状態が残らないように復元する。

## 投石停止

- 効果時間は10秒とする。
- 効果中は新しい投石を開始しない。
- 使用時点で表示中の投石警告を消し、待機中の投石処理をキャンセルする。
- 使用時点ですでに飛んでいる石もすべて消す。
- 効果終了後は、投石間隔を最初から数え直して投石を再開する。
- 効果中に同じアイテムを再使用した場合、停止終了までの時間を10秒へ更新する。

## 確認項目

- `Shift`キーで所持アイテムを使用できる。
- アイテム使用後にインベントリからアイテムがなくなる。
- 王のHPが最大HPの10%回復し、最大値を超えない。
- 傘の見た目と各判定が10秒間2倍になり、終了後に元へ戻る。
- 傘拡大を再使用しても倍率が重複せず、時間だけ更新される。
- 投石停止時に警告、待機中の攻撃、飛行中の石がすべて消える。
- 投石が10秒間発生せず、その後正常に再開する。
- 14店舗のうち販売店舗が6店舗、空店舗が8店舗になる。
- 各アイテムが2店舗ずつで販売される。
- 商品のない店舗で購入操作をしても所持金とインベントリが変化しない。
- UnityのコンパイルエラーとMissing Referenceが発生しない。
